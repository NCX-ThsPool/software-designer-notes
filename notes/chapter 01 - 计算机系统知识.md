## 运算器

### 1. 知识点

运算器负责**数据加工**，常考部件有：**ALU、累加寄存器 AC、数据缓冲寄存器 DR、状态条件寄存器 PSW**。  
其中最容易考的是 **AC** 和 **ALU**：

- **ALU**：真正做算术与逻辑运算
- **AC**：给 ALU 提供工作数据，并暂存运算结果
- **DR**：数据中转站
- **PSW**：保存运算后的状态信息

| 部件 | 作用                         | 高频考法                         |
| ---- | ---------------------------- | -------------------------------- |
| ALU  | 做加减乘除、与或非等逻辑运算 | “属于算术逻辑单元的部件是？”     |
| AC   | 为 ALU 提供数据并暂存结果    | “常用来暂存源操作数和结果的是？” |
| DR   | 数据传送中转                 | “CPU 与内存/外设之间的数据缓冲”  |
| PSW  | 保存条件码和状态             | “保存运算结果状态”               |

### 2. 例题分析

**例 1**  
在 CPU 中，常用来为 ALU 执行算术逻辑运算提供数据并暂存运算结果的寄存器是？  
**思路**：题眼是“为 ALU 提供数据”“暂存运算结果”  
**结论**：选 **累加寄存器 AC**

**例 2**  
属于 CPU 中算术逻辑单元的部件是？  
**思路**：程序计数器、指令寄存器、译码器都偏“控制”；真正做运算的是加法器  
**结论**：选 **加法器**

### 3. 记忆技巧

- 记一句：**“ALU 真运算，AC 先拿后放”**
- 遇到“暂存结果”“给 ALU 提供数据”，优先想到 **AC**

## 控制器

### 1. 知识点

控制器决定计算机运行过程的自动化，负责**取指、分析、发控制信号、处理异常**。  
本部分最常考 4 个点：

1. **控制器产生每条指令的操作信号**
2. **指令 = 操作码 + 地址码**
3. **IR 保存当前指令**
4. **ID 对操作码译码**

| 概念             | 含义                               |
| ---------------- | ---------------------------------- |
| 控制器           | 保证指令正确执行，并能处理异常事件 |
| IR（指令寄存器） | 保存当前正在执行的指令             |
| ID（指令译码器） | 对操作码字段进行分析解释           |
| 操作信号         | 由控制器产生并送往相关部件         |

### 2. 例题分析

**例 1**  
计算机执行指令时，需要由谁产生每条指令的操作信号？  
**结论**：一定是 **CPU 的控制器**

**例 2**  
计算机指令包括操作码和地址码，为分析执行一条指令，两者应存入哪里？  
**思路**：当前正在执行的整条指令都要进入 IR  
**结论**：**操作码和地址码都存入 IR**

**例 3**  
CPU 中译码器的主要作用是什么？  
**结论**：**指令译码**，不是地址译码、数据译码

### 3. 记忆技巧

- 记忆链：**指令进 IR → ID 译码 → 控制器发信号**
- 看到“异常事件”“自动化运行”，优先锁定 **控制器**

## 程序计数器 PC

### 1. 知识点

PC 是控制器中的关键寄存器，作用只有一个：**始终保存下一条将要执行的指令地址**。  
顺序执行时通常 **PC 自动加 1**；转移执行时则改写为转移目标地址。

| 问法                                     | 正确理解 |
| ---------------------------------------- | -------- |
| 跟踪指令地址的寄存器                     | PC       |
| 顺序执行时自动加 1 的寄存器              | PC       |
| 从内存取下一条指令时首先把谁送上地址总线 | PC       |
| 程序计数器属于哪里                       | 控制器   |

### 2. 例题分析

**例 1**  
为实现程序顺序执行，CPU 哪个寄存器中的值自动加 1？  
**结论**：PC

**例 2**  
在一个指令周期中，为从内存读取指令操作码，首先要把哪个寄存器内容送到地址总线？  
**思路**：先给出“下一条指令地址”  
**结论**：PC

**例 3**  
程序计数器 PC 在控制器、运算器、存储器、I/O 设备中属于哪一部分？  
**结论**：**控制器**

### 3. 记忆技巧

- 一句话背熟：**PC 永远指向下一条指令**
- 题里只要出现“下一条指令地址”，答案大概率就是 PC

## 内存按字节编址

### 1. 知识点

按字节编址题本质是**地址差计算题**。  
只要题目没有特别说明，容量计算统一按：

**容量 = 末地址 - 首地址 + 1**

然后再根据单位换算成 B、KB、MB。

### 2. 公式/模板

$\text{容量（字节）} = \text{大地址} - \text{小地址} + 1$

换算关系：

$1\text{KB} = 1024\text{B}$

$1\text{MB} = 1024\text{KB}$

**做题模板**

1. 先把十六进制地址做减法  
2. 结果再加 1  
3. 按题意换算单位  
4. 若问芯片片数，再用“$\text{总容量} \div \text{单片容量}$”

### 3. 例题分析

**例 1**  
地址 0000A000H ～ 0000BFFFH，共有多少个存储单元？  
**解法**：BFFFH - A000H + 1 = 2000H = 8192B = 8KB  
**结论**：8KB

**例 2**  
内存按字节编址，从 A1000H 到 B13FFH 的区域容量是多少？  
**模板**：B13FFH - A1000H + 1，再转 KB

**易错点**

- 忘记 **+1**
- 十六进制减法算完后没有换成十进制/KB

### 4. 记忆技巧

- 口诀：**大减小，再加 1**
- 不要被地址长得吓到，十六进制只是写法不同，本质还是区间长度

## 原码、反码、补码、移码

### 1. 知识点

这是计算机系统高频必考点。  
默认约定：**最高位为符号位，0 正 1 负**。

- **原码**：符号位 + 绝对值
- **反码**：正数不变；负数除符号位外按位取反
- **补码**：正数不变；负数 = 反码 + 1
- **移码**：在补码基础上对符号位取反

补码最重要，因为计算机通常用补码做运算。

| 码制 | 正数           | 负数            | 0 的特点        | 高频结论         |
| ---- | -------------- | --------------- | --------------- | ---------------- |
| 原码 | 与真值接近     | 符号位 + 绝对值 | +0 与 -0 不同   | 直观但不利于运算 |
| 反码 | 与原码相同     | 绝对值按位取反  | +0 与 -0 不同   | 是补码的中间步骤 |
| 补码 | 与原码相同     | 反码 + 1        | 正负 0 编码相同 | **计算机常用**   |
| 移码 | 补码符号位取反 | 补码符号位取反  | 正负 0 编码相同 | 常用于阶码       |

### 2. 公式/模板

$\text{负数补码} = \text{负数反码} + 1$

$\text{补码再求补码} = \text{原码}$

### 3. 例题分析

**例 1**  
16 位补码 FFFFH 表示的十进制数是多少？  
**思路**：全 1 的补码就是 **-1**  
**结论**：-1

**例 2**  
为什么计算机采用补码进行运算？  
**答案要点**：**可以简化运算部件设计**

**例 3**  
若某计算机采用 8 位补码表示整数，哪些运算会溢出？  
**思路**：8 位补码整数范围是 **-128 ~ 127**，结果超出即溢出

### 4. 记忆技巧

- 口诀：**负数看补码，先反后加一**
- 判溢出先看范围，不要一上来就硬算二进制

## 浮点数

### 1. 知识点

浮点数核心只考三件事：

1. **表示形式**：$N = \text{尾数} \times \text{基数}^{\text{指数}}$  
2. **范围由阶码决定，精度由尾数决定**  
3. **浮点数相加先对阶：小阶向大阶对齐，尾数右移**

| 概念         | 结论                          |
| ------------ | ----------------------------- |
| 范围         | 看阶码位数                    |
| 精度         | 看尾数位数                    |
| 对阶原则     | 小阶向大阶对齐                |
| 尾数移动方向 | 右移                          |
| 规格化       | 尾数绝对值通常限制在 [0.5, 1) |

### 2. 公式/模板

$\text{浮点数运算步骤：对阶} \rightarrow \text{尾数运算} \rightarrow \text{结果规格化}$

增加阶码位数：范围增大、精度降低。  

增加尾数位数：精度提高、范围不一定扩大。

### 3. 例题分析

**例 1**  
两个浮点数相加，第一步做什么？  
**结论**：**先统一阶码**

**例 2**  
增加指数部分 E 的位数、减少尾数 M 的位数，会怎样？  
**结论**：**表示范围扩大，精度下降**

**例 3**  
浮点数表示中，哪一部分决定数值范围？  
**结论**：**阶码**

### 4. 记忆技巧

- 记忆句：**范围看阶，精度看尾；先对阶，再动尾**
- 对阶时不是大阶去找小阶，而是 **小阶向大阶靠拢**

## 寻址方式

### 1. 知识点

寻址题本质上考：**操作数在哪里**。  
最常见的 7 种：

- 立即寻址：操作数就在指令中
- 直接寻址：指令给出操作数所在内存地址
- 寄存器寻址：操作数在寄存器中
- 寄存器间接寻址：寄存器里放的是内存地址
- 间接寻址：指令给出的是地址的地址
- 相对寻址：当前地址 + 偏移量
- 变址寻址：变址寄存器内容 + 偏移量

| 寻址方式       | 操作数位置             | 速度特点 |
| -------------- | ---------------------- | -------- |
| 立即寻址       | 指令内部               | 最快     |
| 寄存器寻址     | 寄存器                 | 很快     |
| 直接寻址       | 内存单元               | 次之     |
| 寄存器间接寻址 | 内存，地址在寄存器     | 更慢     |
| 间接寻址       | 内存，且要再找一次地址 | 最慢     |

### 2. 例题分析

**例 1**  
在机器指令地址字段中，直接指出操作数本身的寻址方式称为？  
**结论**：**立即寻址**

**例 2**  
立即寻址、寄存器寻址、直接寻址三者速度比较？  
**结论**：**立即最快，寄存器次之，直接最慢**

### 3. 记忆技巧

- 判断关键：题目问的是“给出操作数本身”，还是“给出地址”
- 一句话：**立即最快，间接最慢**

## 校验码概念

### 1. 知识点

校验码先记住“**码距**”概念。  
码距表示两个合法编码之间至少有多少位不同。

- **码距 = 2**：有检错能力
- **$\text{码距} \geq 3$**：才可能纠错

| 校验方式 | 是否检错 | 是否纠错 | 典型码距 |
| -------- | -------- | -------- | -------- |
| 奇偶校验 | 能       | 不能     | 2        |
| 海明码   | 能       | 能       | 3        |
| CRC      | 能       | 不能     | 常视作 2 |

### 2. 例题分析

**例**  
如果一个编码方案希望既能检错又能纠错，至少要求码距是多少？  
**结论**：至少 **3**

### 3. 记忆技巧

- 就记一句：**检错看 2，纠错至少 3**

---

## 海明码

### 1. 知识点

海明码是既能检错又能纠错的典型编码。  
常考两点：

1. 海明码**可以检错也可以纠错**
2. 若数据位为 n，校验位为 k，则满足：

**$2^k - 1 \geq n + k$**

### 2. 公式/模板

确定海明码校验位个数：
找最小的 $k$，使 $2^k - 1 \geq n + k$

### 3. 例题分析

**例**  
若数据位是 n 位，校验位是 k 位，如何判断 k 是否够用？  
**解法**：代入 **$2^k - 1 \geq n + k$**  
够用则该 k 可行，不够则继续增大 k

### 4. 记忆技巧

- 口诀：**海明能纠错，公式只会这一条**

---

## 循环冗余码 CRC

### 1. 知识点

CRC 常考结论非常单一：

- **可以检错，不能纠错**
- 采用**模 2 运算**
- 数据位 k 后面跟 r 个校验位，总长为 k + r

### 2. 例题分析

**例**  
循环冗余校验码的计算特点是什么？  
**结论**：**模 2 运算**，且**只能检错不能纠错**

### 3. 记忆技巧

- 见到 CRC，先背：**模 2、可检错、不能纠错**

## CISC 和 RISC

### 1. 知识点

这部分最适合用对比表记。  
重点不是背定义，而是能在四个选项里迅速判断哪一项属于 RISC、哪一项属于 CISC。

| 项目       | RISC       | CISC       |
| ---------- | ---------- | ---------- |
| 指令种类   | 少、精简   | 多、丰富   |
| 指令复杂度 | 低         | 高         |
| 指令长度   | 固定       | 可变       |
| 寻址方式   | 少         | 复杂多样   |
| 控制方式   | 硬布线逻辑 | 微程序控制 |
| 通用寄存器 | 多         | 相对少     |
| 流水线     | 更适合采用 | 较弱       |

### 2. 例题分析

**例 1**  
以下关于 RISC 的叙述中，错误的是“指令功能强大、寻址方式复杂多样”。  
**原因**：这明显是 **CISC** 的风格

**例 2**  
以下关于 RISC 和 CISC 的叙述中，正确的是？  
若出现“RISC 采用组合逻辑控制器，CISC 普遍采用微程序控制器”，通常就是正确项

### 3. 记忆技巧

- 口诀：**RISC 少快规整，CISC 多强复杂**
- 遇到“寄存器多、指令短、适合流水”，优先选 **RISC**

## 流水线概念

### 1. 知识点

流水线是把一个重复过程拆成若干子过程，并让它们并行重叠执行。  
最常考的不是长篇定义，而是三个词：

- **操作周期**：取最长段
- **吞吐率**：单位时间完成的结果数
- **加速比**：$\text{不用流水线时间} \div \text{用流水线时间}$

### 2. 公式/模板

操作周期 = 各段中最长执行时间
$\text{加速比} = \text{顺序执行总时间} \div \text{流水线执行总时间}$

### 3. 例题分析

**例**  
某四级流水线各段时间分别为 8ns、9ns、4ns、8ns，则操作周期至少是多少？  
**思路**：周期由最长段决定  
**结论**：9ns

### 4. 记忆技巧

- 先别急着套公式，先找“最长段”

---

## 流水线计算

### 1. 知识点

流水线计算题一般只考 3 个公式。

### 2. 公式/模板

$\text{顺序执行总时间} = \text{单条指令执行时间} \times n$

$\text{流水线执行总时间} = \text{一条指令执行时间} + \text{流水线周期} \times (n - 1)$

$\text{吞吐率} = n \div \text{总执行时间}$

### 3. 例题分析

**例 1**  
若执行 n 条指令，为什么流水线总时间不是 n 倍？  
**原因**：后面的指令可以重叠进入流水线

**例 2**  
如果流水线断流，加速比会怎样？  
**结论**：**明显下降**

### 4. 记忆技巧

- 看到“n 条指令”“几级流水”“每段时间”，立刻写出上面 3 个式子

## 存储器

### 1. 知识点

存储器这块要会分类，不要混。  
最常考：

- **SRAM 构成 Cache**
- **DRAM 构成主存，且需要刷新**
- **虚拟存储器 = 主存 + 辅存**
- **相联存储器按内容访问**

| 名称       | 特点                 | 高频考法                 |
| ---------- | -------------------- | ------------------------ |
| SRAM       | 快、贵               | 构成 Cache               |
| DRAM       | 便宜、容量大、需刷新 | 构成主存                 |
| ROM        | 只读                 | 固化程序                 |
| 闪存       | 掉电不丢失、按块擦除 | 可替代 ROM，不可替代主存 |
| 相联存储器 | 按内容访问           | 不是按地址访问           |

### 2. 例题分析

**例 1**  
哪类存储器构成主存？  
**结论**：**DRAM**

**例 2**  
闪存能否代替主存？  
**结论**：**不能**，因为主存工作特点不同

**例 3**  
相联存储器按什么访问？  
**结论**：**按内容访问**

### 3. 记忆技巧

- 口诀：**主存 DRAM，缓存 SRAM，闪存掉电不丢**

## Cache 概念与关键性质

### 1. 知识点

Cache 是位于 CPU 与主存之间的高速缓冲存储器，本质上是**主存部分内容的副本**。  
目标不是扩大容量，而是**提高访问效率**。

| 判断项                                | 正误 |
| ------------------------------------- | ---- |
| Cache 是主存副本                      | 对   |
| Cache 主要目的是扩大主存容量          | 错   |
| Cache 越大命中率通常越高              | 对   |
| Cache 设置目的是提高 CPU 访问主存效率 | 对   |

### 2. 例题分析

**例 1**  
在 CPU 内外设置多级 Cache，主要目的是？  
**结论**：**提高 CPU 访问主存数据或指令的效率**

**例 2**  
以下关于 Cache 的叙述中，不正确的是“Cache 的设置扩大了主存容量”。  
**原因**：它提升的是速度与命中率，不是主存物理容量

### 3. 记忆技巧

- 一句话：**Cache 不扩容，只提速**

## Cache 与主存映射

### 1. 知识点

Cache 映射题非常高频，结论必须死记：

- **直接映像**：关系固定，冲突最多
- **全相联映像**：最灵活，冲突最少
- **组相联映像**：折中方案
- **Cache 与主存的地址映射由硬件自动完成**

| 映射方式   | 特点                        | 冲突情况 |
| ---------- | --------------------------- | -------- |
| 直接映像   | 简单、固定                  | 最多     |
| 组相联映像 | 折中                        | 较少     |
| 全相联映像 | 任意主存块可放任意 Cache 块 | 最少     |

### 2. 例题分析

**例 1**  
发生块冲突次数最少的是哪种映射？  
**结论**：**全相联映像**

**例 2**  
在程序执行过程中，Cache 与主存的地址映射由谁完成？  
**结论**：**硬件自动完成**

### 3. 记忆技巧

- 口诀：**冲突从多到少：直接 > 组相联 > 全相联**
- 看到“谁负责地址映射”，不犹豫，直接答 **硬件**

## 中断

### 1. 知识点

中断是运行中遇到紧急事件时，CPU 暂停当前程序，转去处理中断服务程序，处理完再返回原程序。  
本部分要抓 4 句话：

1. **中断向量提供中断服务程序入口地址**
2. **中断响应时间：从发出请求到进入中断服务程序**
3. **保存现场：为了正确返回原程序继续执行**
4. **中断与异常不同：外部请求通常叫中断，处理器内部特殊事件常叫异常**

| 概念             | 正确理解                   |
| ---------------- | -------------------------- |
| 中断向量         | 中断服务程序入口地址       |
| 保存现场         | 为了返回原程序             |
| 中断响应时间     | 请求发出到进入中断处理程序 |
| 多级中断保护现场 | 用堆栈最有效               |

### 2. 例题分析

**例 1**  
中断向量提供什么？  
**结论**：**中断服务程序入口地址**

**例 2**  
中断处理中保存现场的目的是什么？  
**结论**：**能正确返回到被中断程序继续执行**

**例 3**  
DMA 传送结束与除数为 0 分别属于什么？  
**结论**：前者通常看作**中断**，后者是**异常**

### 3. 记忆技巧

- 口诀：**向量给入口，现场为返回**
- 中断题很爱考“保存现场”，千万别误选“给中断程序提供数据”

## DMA

### 1. 知识点

DMA 是 Direct Memory Access，直接存储器访问。  
它的核心思想是：**主存和外设之间直接成块传送数据，CPU 只在开始和结束时干预较多**。

| 方式     | CPU 与外设能否并行 | 传送单位 | 谁把数据送入内存 |
| -------- | ------------------ | -------- | ---------------- |
| 程序查询 | 不能               | 字       | CPU              |
| 中断方式 | 能                 | 字       | CPU              |
| DMA      | 能                 | 块       | 外设/DMA 控制器  |

### 2. 例题分析

**例 1**  
DMA 控制方式是在谁和谁之间直接建立数据通路？  
**结论**：**主存与外设**

**例 2**  
采用 DMA 方式传送数据时，每传送一个数据通常占用什么周期？  
**结论**：常考 **总线周期**

**例 3**  
哪种 I/O 控制方式传输速度快、CPU 干预少？  
**结论**：**DMA**

### 3. 记忆技巧

- 口诀：**DMA 成块传，主存外设直接连**
- 遇到“并行工作 + 块传输”，优先 DMA

## 总线

### 1. 知识点

总线是连接多个部件的一组公共信号线。  
三大总线一定要分清：

- **数据总线 DB**：传数据，**双向**
- **地址总线 AB**：传地址，**单向**
- **控制总线 CB**：传控制和状态，整体看双向

| 总线     | 作用               | 方向     | 决定什么         |
| -------- | ------------------ | -------- | ---------------- |
| 数据总线 | 传送数据信息       | 双向     | 一次交换数据位数 |
| 地址总线 | 传送地址           | 单向     | 最大寻址能力     |
| 控制总线 | 传送控制/状态/时序 | 整体双向 | 系统控制能力     |

### 2. 例题分析

**例 1**  
地址总线宽度决定什么？  
**结论**：**最大寻址能力**

**例 2**  
数据总线是单向还是双向？  
**结论**：**双向**

### 3. 记忆技巧

- 口诀：**数据双向、地址单向、控制管全场**

## 公钥与私钥

### 1. 知识点

非对称加密也叫公钥体系。  
最容易混的只有两句：

- **用接收方公钥加密，用接收方私钥解密**
- **公钥加密是为了保密，私钥签名是为了认证/不可否认**

### 2. 例题分析

**例**  
公钥加密算法也称什么？  
**结论**：**非对称加密算法**

### 3. 记忆技巧

- 记忆：**保密看接收方，签名看发送方**

---

## 数字签名

### 1. 知识点

数字签名不是“给消息加密”那么简单，它主要解决：

- 消息是否被伪造
- 发送方能否否认

标准过程：

1. 发送方先对消息摘要做处理
2. **用发送方私钥签名**
3. 接收方**用发送方公钥验证**

| 目的           | 是否支持         |
| -------------- | ---------------- |
| 验证消息真实性 | 支持             |
| 防止发送方否认 | 支持             |
| 保密明文内容   | 不是它的主要职责 |

### 2. 例题分析

**例**  
数字签名的验证过程是什么？  
**结论**：**发送方私钥签名，发送方公钥验证**

### 3. 记忆技巧

- 口诀：**私钥签，公钥验，不可否认最关键**

## 数字证书

### 1. 知识点

数字证书解决的是“**你到底是不是你**”。  
核心角色是 **CA（证书授权中心）**。

标准过程：

1. 用户把身份信息和公钥提交给 CA  
2. CA 颁发数字证书  
3. **数字证书由 CA 私钥签名**
4. 用户通过 **CA 公钥验证** 证书真伪

| 对象     | 做什么                 |
| -------- | ---------------------- |
| CA 私钥  | 给证书签名             |
| CA 公钥  | 验证证书真伪           |
| 用户公钥 | 存在证书中，供他人获取 |

### 2. 例题分析

**例 1**  
某网站向 CA 申请了数字证书，用户通过什么验证什么来确定网站合法性？  
**结论**：**用 CA 的公钥验证 CA 的签名**

**例 2**  
确认用户身份通常使用什么？  
**结论**：**数字证书**

### 3. 记忆技巧

- 口诀：**CA 私钥签，CA 公钥验**
- 不要写反成“CA 私钥验证”

## 系统可靠度

### 1. 知识点

可靠性题只要分清**串联**和**并联**就够了。

### 2. 公式/模板

串联系统：

$R = R_1 \times R_2 \times \cdots \times R_n$

并联系统：

$R = 1 - (1 - R_1)(1 - R_2)\cdots(1 - R_n)$

### 3. 例题分析

**例 1**  
系统中所有部件都正常系统才正常，这是哪种结构？  
**结论**：**串联**

**例 2**  
只要有一个部件正常系统就能工作，这是哪种结构？  
**结论**：**并联**

### 4. 记忆技巧

- 口诀：**串联相乘，并联减失败**

## 1. 运算器、控制器、PC 相关原题与解析

### 题 1

**原题**：计算机执行指令的过程中，需要由（A）产生每条指令的操作信号并将信号送往相应的部件进行处理，以完成指定的操作。（2019年上半年）  
A. CPU 的控制器  B. CPU 的运算器  C. DMA 控制器  D. Cache 控制器

**解析**：题眼是“产生操作信号”。运算器负责算术逻辑运算，DMA 控制器负责高速 I/O 传输控制，Cache 控制器负责缓存管理；真正统一调度各部件、发出控制信号的是 CPU 控制器。  
**答案方向**：A，CPU 的控制器。

### 题 2

**原题**：在 CPU 中，常用来为 ALU 执行算术逻辑运算提供数据并暂存运算结果的寄存器是（D）。（2014年上半年）  
A. 程序计数器  B. 状态寄存器  C. 通用寄存器  D. 累加寄存器

**解析**：题眼有两个：“给 ALU 提供数据”“暂存结果”。这正是累加寄存器 AC 的典型职责。  
**答案方向**：D，累加寄存器。

### 题 3

**原题**：在 CPU 中，（B）不仅要保证指令的正确执行，还要能够处理异常事件。（2012年下半年）  
A. 运算器  B. 控制器  C. 寄存器组  D. 内部总线

**解析**：遇到“正确执行”“处理异常”这类描述，优先想到控制器。  
**答案方向**：B，控制器。

### 题 4

**原题**：计算机指令一般包括操作码和地址码两部分，为分析执行一条指令，其（C）。（2010年上半年）  
A. 操作码应存入 IR，地址码应存入 PC  
B. 操作码应存入 PC，地址码应存入 IR  
C. 操作码和地址码都应存入 IR  
D. 操作码和地址码都应存入 PC

**解析**：IR 存放当前正在执行的整条指令，既包含操作码，也包含地址码；PC 只负责给出下一条指令地址。  
**答案方向**：C，操作码和地址码都存入 IR。

### 题 5

**原题**：编写汇编语言程序时，下列寄存器中程序员可访问的是（5）。（2010年下半年）

**解析**：这类题通常考“对用户透明”和“程序员可见”。像 IR、ID 这类内部控制寄存器对程序员通常不可直接访问；通用寄存器、程序状态字等更可能可访问。  
**解题抓手**：凡是“控制器内部暂存当前指令”的寄存器一般对程序员透明；凡是“用于编程操作”的寄存器通常可访问。  
**答案方向**：选择“程序员可见寄存器”。

### 题 6

**原题**：指令寄存器的位数取决于（2）。（2013年下半年）

**解析**：IR 用来保存当前指令，所以它的长度必须至少能装下一条完整指令。  
**答案方向**：取决于**指令字长**。

### 题 7

**原题**：为实现程序顺序执行，CPU 中（PC）的内容通常自动加 1；若发生转移，则改写为转移目标地址。（综合考点）

**解析**：PC 永远保存“下一条将要执行的指令地址”，这是最稳定的考法。  
**答案方向**：程序计数器 PC。

## 2. 数据表示、补码、浮点数原题与解析

### 题 8

**原题**：16 位补码 FFFFH 表示的十进制数是（ ）。

**解析**：全 1 的补码是补码题中的经典结论。8 位 11111111 表示 -1，16 位 FFFFH 本质上也是全 1。  
**答案方向**：-1。

### 题 9

**原题**：计算机采用补码表示数据的主要原因是（ ）。

**解析**：并不是因为“表示范围最大”或者“最直观”，而是因为补码使加减法统一，硬件实现简单。  
**答案方向**：**简化计算机运算部件设计**。

### 题 10

**原题**：若计算机存储数据采用双符号位，两个符号相同的数相加时，如果运算结果的两个符号位经（3）运算得 1，则可断定结果溢出。（2013年下半年）

**解析**：双符号位判溢出时，看两个符号位是否不同；“相同为 0，不同为 1”的逻辑是异或。  
**答案方向**：逻辑异或。

### 题 11

**原题**：浮点数中，阶码的位数增加而尾数位数减少，则（ ）。

**解析**：阶码决定表示范围，尾数决定精度。所以“阶多尾少”意味着范围变大、精度变差。  
**答案方向**：**表示范围扩大，精度降低**。

### 题 12

**原题**：浮点数加减运算时，第一步应进行（ ）。

**解析**：先统一阶码，才能对尾数做运算。并且是小阶向大阶对齐，尾数右移。  
**答案方向**：对阶。

### 题 13

**原题**：一般浮点数表示中，决定数值范围的是（ ），决定精度的是（ ）。

**解析**：范围看阶码，精度看尾数，这是浮点题的固定结论。  
**答案方向**：范围看阶码；精度看尾数。

### 题 14

**原题**：下列关于逻辑同或和异或的叙述中，正确的是（ ）。

**解析**：异或是“相同为 0，不同为 1”；同或则相反，是“相同为 1，不同为 0”。  
**答案方向**：抓住这两个定义即可排除干扰项。

## 3. 寻址方式原题与解析

### 题 15

**原题**：在机器指令地址字段中，直接指出操作数本身的寻址方式称为（ ）。

### 题 16

### 题 19

## 4. 海明码、CRC、校验码原题与解析

### 题 20

**原题**：海明校验码是在 n 个数据位之外增设 k 个校验位，从而形成一个 n+k 位的新码字，使新的码字码距拉大。n 与 k 的关系是（1）。（2009年上半年）

**解析**：海明码最核心的公式只有一个：  
$2^k - 1 \geq n + k$

做题时直接代入，找满足条件的最小 k。  
**答案方向**：$2^k - 1 \geq n + k$。

### 题 21

**原题**：海明码利用奇偶性检错和纠错，通过在 n 个数据位之间插入 k 个校验位。若 n=48，则 k 至少为（3）。（2014年上半年）

**解析**：代入公式 $2^k - 1 \geq 48 + k$。  
试算：  
k=5，31 < 53，不够；  
$k=6$，$63 \geq 54$，够。  
**答案方向**：6 位。

### 题 22

**原题**：已知数据信息为 16 位，最少应附加（4）位校验位，以实现海明码纠错。（2016年下半年）

**解析**：代入公式 $2^k - 1 \geq 16 + k$。  
k=4，15 < 20；  
$k=5$，$31 \geq 21$。  
**答案方向**：5 位。

### 题 23

**原题**：已知数据信息为 16 位，最少应附加（5）位校验位，才能实现海明码纠错。（2017年上半年）

**解析**：与上一题完全同型。  
**答案方向**：5 位。

### 题 24

**原题**：以 10 位数据为例，其海明码中某一数据位由哪些校验位负责校验。（2018年上半年）

**解析**：海明码定位法看“位序号的二进制展开”。一个数据位由哪些校验位校验，就看它的位序号中哪些二进制位为 1。  
**答案方向**：按“位序号二进制拆分”判断。

### 题 25

**原题**：循环冗余校验码 CRC 利用生成多项式进行编码。设数据位为 k，校验位为 r，则 CRC 码格式为（2）。（2012年下半年）

**解析**：CRC 左边是原始信息码，右边是校验码，总长度是 $k+r$。  
**答案方向**：由 **k 位信息位 + r 位校验位** 组成。

### 题 26

**原题**：在（5）校验方法中，采用模 2 运算来构造校验位。（2019年上半年）

**解析**：模 2 运算是 CRC 最典型的辨识词。  
**答案方向**：CRC。

### 题 27

**原题**：奇偶校验、海明码、CRC 三者中，能纠错的是（ ）。

**解析**：奇偶校验只能检错，CRC 也主要检错；能纠错的是海明码。  
**答案方向**：海明码。

## 5. RISC / CISC、流水线原题与解析

### 题 28

**原题**：以下关于 CISC 和 RISC 的叙述中，错误的是（2）。（2009年下半年）

**解析**：这类题最稳的做法不是硬背句子，而是套对比表：  

- RISC：少、短、固定、寄存器多、适合流水  
- CISC：多、复杂、可变长、寻址丰富  
  若某选项把这两类特征对调，基本就是错项。  
  **答案方向**：找“RISC 却说成复杂、寻址多样”的选项。

### 题 29

**原题**：（5）不是 RISC 的特点。（2013年下半年）

**解析**：凡是“指令复杂”“寻址方式复杂多样”“更多采用微程序控制”的表述，都更偏 CISC。  
**答案方向**：排除 RISC 的典型特征，选 CISC 风格表述。

### 题 30

**原题**：以下关于 RISC 和 CISC 的叙述中，不正确的是（5）。（2014年下半年）

**解析**：同型题，仍然用对比表判定。  
**答案方向**：抓“少而规整”与“多而复杂”的对照。

### 题 31

**原题**：CISC 是（6）的简称。（2015年下半年）

**解析**：记英文全称即可。  
**答案方向**：Complex Instruction Set Computer，复杂指令集计算机。

### 题 32

**原题**：以下关于 RISC 技术的叙述中，错误的是（6）。（2019年上半年）

**解析**：若出现“指令功能丰富、长度可变、寻址复杂”，那就是错误项。  
**答案方向**：选择不符合 RISC 基本风格的说法。

### 题 33

**原题**：某四级指令流水线各段时间分别为 8ns、9ns、4ns、8ns，则操作周期至少为（2）ns。（2017年下半年）

**解析**：流水线周期取最长段，而不是平均值，也不是总和。最长的是 9ns。  
**答案方向**：9ns。

### 题 34

**原题**：流水线的吞吐率是单位时间内流水线处理任务数。如果各段流水操作时间不同，则吞吐率是（6）的倒数。（2018年上半年）

**解析**：瓶颈决定速度，最长流水段限制吞吐率。  
**答案方向**：最长流水段操作时间的倒数。

### 题 35

**原题**：下列关于流水线方式执行指令的叙述中，不正确的是（6）。（2018年下半年）

**解析**：常见错项一般有三类：  

1. 把周期说成最短段时间；  
2. 认为一定不需等待；  
3. 把吞吐率理解成单条指令执行时间的倒数。  
   **答案方向**：找违背“周期取最长段”的表述。

### 题 36

**原题**：一条指令分为取指、分析、执行三步，若三步时间分别为 t1、t2、t3，按顺序执行和按流水线执行各需多少时间。（2012年上半年）

**解析**：  

- 顺序执行 $n$ 条：$n \times (t_1+t_2+t_3)$  
- 流水线执行 $n$ 条：$\text{一条完整时间} + \text{周期} \times (n-1)$  
  其中周期为 max(t1,t2,t3)  
  **答案方向**：直接代公式计算。

### 题 37

**原题**：执行完 100 条指令，在给定取指、分析、执行时间的流水线条件下，总时间为（5）。（2016年下半年）

**解析**：标准流水线计算题，不要逐条数，直接用：  
$\text{总时间} = \text{首条指令总时间} + \text{周期} \times 99$  
**答案方向**：套入首条总时长和最长段周期。

### 题 38

**原题**：从头到尾执行完 500 条指令需多少时间。（2019年下半年）

**解析**：与上题同型。大量指令时，后续部分由周期主导。  
**答案方向**：按流水线总时间公式求。

## 6. 存储器、Cache、主存映射原题与解析

### 题 39

**原题**：（3）是指按内容访问的存储器。（2009年上半年）

**解析**：按内容访问而不是按地址访问，这是相联存储器的核心定义。  
**答案方向**：相联存储器。

### 题 40

**原题**：常用的虚拟存储器由（1）两级存储器组成。（2013年上半年）

**解析**：虚拟存储器并不是 Cache + 主存，而是主存与辅存配合形成的逻辑大容量存储。  
**答案方向**：主存 + 辅存。

### 题 41

**原题**：计算机系统的主存主要是由（4）构成的。（2017年下半年）

**解析**：主存重容量和成本，通常由 DRAM 实现；Cache 则多用 SRAM。  
**答案方向**：DRAM。

### 题 42

**原题**：（5）是一种需要通过周期性刷新来保持数据的存储器件。（2021年下半年）

**解析**：需要刷新是 DRAM 的典型特征。  
**答案方向**：DRAM。

### 题 43

**原题**：以下关于闪存 Flash Memory 的叙述中，错误的是（4）。（2021年上半年）

**解析**：闪存掉电不丢失、可按块擦除、可替代部分 ROM 用途；但它不能直接替代主存。  
**答案方向**：选择“可替代主存”之类的错误表述。

### 题 44

**原题**：CPU 访问存储器时，被访问数据一般聚集在一个较小连续区域中。若一个单元已被访问，则其邻近单元有可能还要被访问，该特性称为（3）。（2019年上半年）

**解析**：这是程序局部性中的空间局部性。  
**答案方向**：空间局部性。

### 题 45

**原题**：以下关于 Cache 的叙述中，正确的是（6）。（2009年下半年）

**解析**：Cache 内容是主存副本；作用是提高访问速度；对程序员通常透明。只要有一项说“扩大主存容量”，通常就是错项。  
**答案方向**：选“用于提高访问速度、内容为主存副本”的表述。

### 题 46

**原题**：Cache 的地址映像方式中，发生块冲突次数最小的是（3）。（2015年上半年）

**解析**：冲突大小规律：  
直接映像 > 组相联映像 > 全相联映像  
所以冲突最少的是全相联。  
**答案方向**：全相联映像。

### 题 47

**原题**：主存与 Cache 的地址映射方式中，（2）方式可以实现主存任意一块装入 Cache 任意位置，只有装满才需要替换。（2016年上半年）

**解析**：能“任意放”的只有全相联映像。  
**答案方向**：全相联映像。

### 题 48

**原题**：在程序执行过程中，Cache 与主存的地址映像由（3）完成。（2011年下半年）

**解析**：这是固定考点，不由操作系统、不由程序员，而是硬件自动完成。  
**答案方向**：硬件。

### 题 49

**原题**：位于 CPU 与主存之间的高速缓冲存储器用于存放部分主存数据的拷贝，主存地址与 Cache 地址之间的转换工作由（1）完成。（2012年上半年）

**解析**：同型题。  
**答案方向**：硬件自动完成。

### 题 50

**原题**：在程序执行过程中，高速缓存（Cache）与主存间的地址映射是由（1）完成的。（2020年下半年）

**解析**：仍是硬件完成。  
**答案方向**：硬件。

### 题 51

**原题**：在 CPU 内外常需设置多级高速缓存，主要目的是（1）。（2019年下半年）

**解析**：不是扩大容量，而是缩短 CPU 访问主存平均时间。  
**答案方向**：提高 CPU 访问数据和指令的效率。

## 7. 中断、DMA、I/O 方式原题与解析

### 题 52

**原题**：中断向量可提供（2）。（2013年上半年）

**解析**：中断向量就是中断服务程序的入口地址。  
**答案方向**：中断服务程序入口地址。

### 题 53

**原题**：计算机中 CPU 的中断响应时间指的是（4）的时间。（2015年上半年）

**解析**：不是整个中断服务全过程，而是“从提出中断请求到进入中断服务程序”的这段时间。  
**答案方向**：中断请求发出到进入中断服务程序。

### 题 54

**原题**：中断处理中保存现场的目的是（3）。（2019年下半年）

**解析**：保存现场并不是为了“让中断程序执行得更快”，而是为了处理结束后能正确回到原程序继续。  
**答案方向**：正确返回被中断程序继续执行。

### 题 55

**原题**：为了便于实现多级中断嵌套，使用（3）来保护断点和现场最有效。（2013年上半年）

**解析**：多级嵌套时，现场保存最适合用栈结构。  
**答案方向**：堆栈。

### 题 56

**原题**：以下关于计算机系统中断概念的叙述中，正确的是（3）。（2010年上半年）

**解析**：这类题抓三点：  

- 中断是暂停当前程序去处理紧急事件  
- 处理完要返回原程序  
- 中断向量给入口地址  
  **答案方向**：选择符合上述三个特征的说法。

### 题 57

**原题**：异常是处理器内部特殊事件，中断是来自处理器外部的请求事件。以下叙述中正确的是（6）。（2021年上半年）

**解析**：常见考法是让考生区分“除数为 0”“缺页”“I/O 完成”“DMA 结束”等属于异常还是中断。  
**答案方向**：处理器内部错误/事件偏异常，外设请求偏中断。

### 题 58

**原题**：在输入输出控制方法中，采用（1）可以使设备与主存间的数据块传送无需 CPU 干预。（2010年下半年）

**解析**：块传输、CPU 少干预，这是 DMA 的专属关键词。  
**答案方向**：DMA 方式。

### 题 59

**原题**：I/O 设备与主机间进行数据传输时，CPU 只需在开始和结束时少量处理，无需干预数据传送过程的是（5）方式。（2012年下半年）

**解析**：同型题。  
**答案方向**：DMA 方式。

### 题 60

**原题**：DMA 工作方式下，在（4）之间建立了直接的数据通路。（2013年上半年）

**解析**：不是 CPU 与外设直接连，而是主存与外设建立直接数据通路。  
**答案方向**：主存与外设。

### 题 61

**原题**：采用 DMA 方式传送数据时，每传送一个数据都需要占用一个（3）。（2021年上半年）

**解析**：DMA 传输仍要占用总线。题目通常考“总线周期”。  
**答案方向**：总线周期。

### 题 62

**原题**：计算机运行过程中，CPU 需要与外设进行数据交换。采用（6）控制技术时，CPU 与外设可并行工作。（2017年下半年）

**解析**：中断方式和 DMA 方式都可并行，但若题眼包含“块传输、少干预”，更偏 DMA。若题目仅说“可并行工作”，则需结合选项。  
**答案方向**：优先判断中断方式或 DMA 方式，排除程序查询。

## 8. 总线原题与解析

### 题 63

**原题**：处理机主要由处理器、存储器和总线组成，总线包括（4）。（2009年上半年）

**解析**：总线固定分三类：数据总线、地址总线、控制总线。  
**答案方向**：数据总线 + 地址总线 + 控制总线。

### 题 64

**原题**：若内存容量为 4GB，字长为 32，则（3）。（2010年下半年）

**解析**：这类题通常考地址总线宽度、数据总线宽度或编址单位。4GB 若按字节编址，需要 32 位地址空间。  
**答案方向**：结合“$4\text{GB}=2^{32}\text{字节}$”推理。

### 题 65

**原题**：在计算机系统中采用总线结构，便于实现系统的积木化构造，同时可以（3）。（2011年上半年）

**解析**：总线优点是减少连线数量、便于扩展、提高模块化水平。  
**答案方向**：减少信息传输线数量，便于扩展与维护。

### 题 66

**原题**：总线复用方式可以（4）。（2011年下半年）

**解析**：所谓复用，是一组线在不同时间承担不同功能，其主要效果是减少连线数。  
**答案方向**：减少总线线数、降低成本。

### 题 67

**原题**：（6）不属于系统总线。（2012年下半年）

**解析**：系统总线看三类 DB/AB/CB。像 PCI、USB、SCSI 往往是局部总线或外设总线。  
**答案方向**：选不属于 DB/AB/CB 的选项。

### 题 68

**原题**：三总线结构的计算机总线系统由（1）组成。（2014年下半年）

**解析**：若考“三总线结构”，一般说的是存储器总线、I/O 总线、直接内存访问总线，具体看教材表述；若题目在微机基础里，则常规仍回到数据/地址/控制。  
**答案方向**：按该题选项所属教材口径判断，但核心仍是“分工明确”。

### 题 69

**原题**：总线宽度 32bit，时钟频率 200MHz，若每 5 个时钟周期传送一个 32bit 字，则带宽为（5）MB/s。（2015年上半年）

**解析**：  
每秒传送次数 = 200M / 5 = 40M 次  
每次 32bit = 4B  
$\text{带宽} = 40\text{M} \times 4\text{B} = 160\text{MB/s}$  
**答案方向**：160MB/s。

---

- 第一步：理解已知条件

| 参数     | 值                    | 含义                                        |
| -------- | --------------------- | ------------------------------------------- |
| 总线宽度 | 32 bit                | 一次能传输的数据量 = 4 字节                 |
| 时钟频率 | 200 MHz               | 每秒钟有 200M 个时钟周期                    |
| 传输节奏 | 每 5 个时钟周期传一次 | 不是每个周期都传数据，而是 5 个周期才传一次 |

> 注意：带宽 = 单位时间内传输的总数据量（单位：MB/s 或 bit/s）

---

- 第二步：求每秒传输次数

时钟频率 200 MHz 表示：**1 秒内有 \( 200 \times 10^6 \) 个时钟周期**

每 5 个时钟周期传输 1 次数据，每秒传输次数为：

$$
\text{每秒传输次数} = \frac{200\ \text{MHz}}{5} = 40\ \text{M 次/秒}
$$

其中 \( \text{M} = 10^6 \)，即 \( 40 \times 10^6 \) 次/秒。

---

- 第三步：求每次传输的数据量

总线宽度为 32 bit，每次传输一个 32 bit 的字：

$$
\text{每次传输数据量} = \frac{32\ \text{bit}}{8\ \text{bit/B}} = 4\ \text{B}
$$

---

- 第四步：计算带宽

$$
\begin{aligned}
\text{带宽} &= \text{每秒传输次数} \times \text{每次传输数据量} \\[4pt]
&= (40 \times 10^6\ \text{次/秒}) \times (4\ \text{B}) \\[4pt]
&= 160 \times 10^6\ \text{B/s} \\[4pt]
&= 160\ \text{MB/s}
\end{aligned}
$$

---

- 通用公式（可直接套用）

$$
\boxed{\text{带宽 (MB/s)} = \frac{\text{时钟频率 (MHz)}}{\text{节拍数}} \times \frac{\text{总线宽度 (bit)}}{8}}
$$

代入本题数据：

$$
\text{带宽} = \frac{200}{5} \times \frac{32}{8} = 40 \times 4 = 160\ \text{MB/s}
$$

---

**答案方向**：160 MB/s



### 题 70

**原题**：以下关于总线的叙述中，不正确的是（6）。（2016年上半年）

**解析**：常见错项是把地址总线说成双向，或把数据总线说成决定寻址能力。  
**答案方向**：抓住三总线职责分工即可。

### 题 71

**原题**：以下关于 PCI 总线和 SCSI 总线的叙述中，正确的是（2）。（2021年下半年）

**解析**：PCI 多作高速并行内总线；SCSI 是小型计算机系统接口，典型并行外总线。  
**答案方向**：选“PCI 属于内总线、SCSI 属于外总线”这一类表述。

## 9. 公钥、签名、证书、算法原题与解析

### 题 72

**原题**：网络安全包含可用性、保密性、完整性和通信对象真实性。其中，数字签名是对（8）的保护。（2009年上半年）

**解析**：数字签名主要确保来源真实、不可否认，也能辅助完整性校验，但不是做保密。  
**答案方向**：真实性/不可否认性。

### 题 73

**原题**：甲和乙要进行通信，甲对发送的消息附加了数字签名，乙收到该消息后利用（7）验证真实性。（2012年上半年）

**解析**：签名是甲用自己的私钥完成，验证就必须用甲的公钥。  
**答案方向**：发送方甲的公钥。

### 题 74

**原题**：（8）不是数字签名的作用。（2016年下半年）

**解析**：数字签名不以“保密明文”为核心目的。  
**答案方向**：凡是“对消息进行保密/加密传输”的表述通常不是签名作用。

### 题 75

**原题**：在安全通信中，S 将信息使用（8）进行数字签名，T 收到后利用（9）验证其真实性。（2018年上半年）

**解析**：固定搭配：  

- 发送方私钥签名  
- 发送方公钥验证  
  **答案方向**：私钥签名，公钥验证。

### 题 76

**原题**：某网站向 CA 申请了数字证书，用户通过（9）来验证网站真伪。（2009年下半年）

**解析**：验证网站证书真伪，本质是验证 CA 的签名，因此用 CA 公钥。  
**答案方向**：CA 的公钥。

### 题 77

**原题**：用户 A 从 CA 获得用户 B 的数字证书，并利用（7）验证数字证书的真实性。（2011年上半年）

**解析**：证书是 CA 签发的，验证者用 CA 公钥。  
**答案方向**：CA 的公钥。

### 题 78

**原题**：从认证中心 CA 获取用户 B 的数字证书，该证书用（9）作数字签名；从数字证书中可以获得 B 的公钥。（2011年下半年）

**解析**：签发证书的一方是 CA，因此签名密钥是 CA 私钥。  
**答案方向**：CA 的私钥。

### 题 79

**原题**：用户 B 收到用户 A 带数字签名的消息 M，为验证 M 的真实性，首先从 CA 获取用户 A 的数字证书，并利用（8）验证证书真伪，然后利用（9）验证 M 的真实性。（2012年下半年）

**解析**：分两步：  

1. 证书真伪 → CA 公钥  
2. 消息签名真伪 → A 的公钥  
   **答案方向**：先 CA 公钥，再 A 的公钥。

### 题 80

**原题**：PKI 体制中，保证数字证书不被篡改的方法是（8）。（2013年下半年）

**解析**：数字证书之所以不能被随意改，是因为其中包含 CA 的数字签名。  
**答案方向**：CA 对证书进行数字签名。

### 题 81

**原题**：用户身份认证可使用（9）；消息不可否认可使用（10）。（2019年上半年）

**解析**：身份认证靠数字证书，消息不可否认靠数字签名。  
**答案方向**：数字证书；数字签名。

### 题 82

**原题**：某电子商务网站向 CA 申请了数字证书，用户可以通过使用（10）验证（11）的真伪来确定网站合法性。（2019年下半年）

**解析**：这类题常写成“使用 CA 公钥验证 CA 签名”。  
**答案方向**：CA 公钥；CA 的签名/证书签名。

### 题 83

**原题**：下列算法中，不属于公开密钥加密算法的是（9）。（2013年下半年）

**解析**：非对称算法常见有 RSA、ECC、DSA；DES、AES、IDEA、RC4 这类都是对称算法。  
**答案方向**：选对称加密算法。

### 题 84

**原题**：可用于数字签名的算法是（7）。（2016年下半年）

**解析**：可签名的算法常见有 RSA、DSA、ECC 相关算法。  
**答案方向**：选非对称签名算法。

### 题 85

**原题**：以下加密算法中适合对大量明文消息进行加密传输的是（8）。（2017年上半年）

**解析**：大量数据传输优先用对称加密，因为快。  
**答案方向**：AES / DES / 3DES 这类对称算法。

### 题 86

**原题**：DES 是（7）算法。（2018年下半年）

**解析**：DES 属于对称密钥加密。  
**答案方向**：对称加密算法。

### 题 87

**原题**：MD5 是（9）算法，对任意长度输入计算得到的结果长度为（10）位。（2018年下半年）

**解析**：MD5 是摘要算法，固定输出 128 位。  
**答案方向**：摘要算法；128 位。

### 题 88

**原题**：下列算法中，不属于公开密钥加密算法的是（8）。（2019年下半年）

**解析**：与前面同型，排除 RSA/ECC/DSA，选 DES/AES 等。  
**答案方向**：对称算法。

### 题 89

**原题**：以下关于认证和加密的叙述中，错误的是（7）。（2020年下半年）

**解析**：常见错项：  

- 把签名和加密混为一谈  
- 把 CA 公钥/私钥用反  
- 把“公钥加密、私钥解密”和“私钥签名、公钥验证”写反  
  **答案方向**：抓这三处反向错误。

### 题 90

**原题**：下列算法中属于非对称加密算法的是（9）。（2021年上半年）

**解析**：非对称算法代表就是 RSA、ECC、DSA。  
**答案方向**：RSA/ECC/DSA 类。

### 题 91

**原题**：AES 是一种（8）算法。（2021年下半年）

**解析**：AES 是高级加密标准，典型对称加密算法。  
**答案方向**：对称加密算法。

## 10. 可靠性原题与解析

### 题 92

**原题**：某计算机系统由若干部件构成，假定每个部件千小时可靠度都为 R，则系统千小时可靠度为（2）。（2010年上半年）

**解析**：先看结构图。  

- 若所有部件都正常系统才正常 → 串联，可靠度相乘  
- 若有冗余，只要一个支路正常即可 → 并联，1 减全失败  
  **答案方向**：按串并联系统公式计算。

### 题 93

**原题**：某系统由冗余部件构成，每个部件可靠度为 R，则系统可靠度为（4）。（2017年上半年）

**解析**：冗余说明至少有并联结构。  
**答案方向**：优先使用并联系统公式：$1 - (1-R_1)(1-R_2)\cdots$

### 题 94

**原题**：某系统由 3 个部件构成，每个部件千小时可靠度都为 R，该系统千小时可靠度为某表达式，则该系统构成方式是（4）。（2019年上半年）

**解析**：这类题是“公式反推结构图”。  

- 若结果是 $R_1R_2R_3$，说明串联  
- 若结果是 $1-(1-R)^n$，说明并联  
- 若混合式，则说明串并联混合  
  **答案方向**：根据目标表达式反推结构。

### 题 95

**原题**：某系统中部件 2、3 为冗余系统，要求整体可靠度不小于 0.85，则部件 4 可靠度至少应为（2）。（2019年下半年）

**解析**：先算冗余子系统可靠度，再与串联部分相乘，最后反解未知量。  
**答案方向**：  

1. 冗余部分：1-(1-r2)(1-r3)  
2. 再乘其余串联系统  
3. 解不等式得到部件 4 最小值

## 11. 其他零散原题与解析

### 题 96

**原题**：计算机采用分级存储体系的主要目的是为了解决（2）问题。（2014年下半年）

**解析**：分层存储并不是只为扩大容量，也不是只为降低成本，而是平衡速度、容量、成本三者矛盾。  
**答案方向**：解决**存储容量、成本和速度之间的矛盾**。

### 题 97

**原题**：CPU 对其访问速度最快的是（2）。（2015年上半年）

**解析**：通常访问层次中，寄存器 > Cache > 主存 > 外存。  
**答案方向**：寄存器。

### 题 98

**原题**：安全需求可划分为物理线路安全、网络安全、系统安全和应用安全。属于系统安全的是（8），属于应用安全的是（9）。（2015年上半年）

**解析**：  

- 系统安全：漏洞补丁、访问控制、OS 安全  
- 应用安全：数据库安全、业务逻辑安全  
- 网络安全：防火墙、入侵检测  
- 物理安全：机房、电源  
  **答案方向**：按这套分类判断。

### 题 99

**原题**：（7）不属于主动攻击。（2015年下半年）

**解析**：主动攻击会修改、伪造、阻断；被动攻击侧重窃听和分析。  
流量分析、会话拦截通常属于被动攻击。  
**答案方向**：选被动攻击项。

### 题 100

**原题**：下列攻击行为中，属于典型被动攻击的是（9）。（2017年下半年）

**解析**：被动攻击典型例子就是流量分析、通信监听、会话拦截。  
**答案方向**：选“流量分析/监听”类表述。

### 题 101

**原题**：VLIW 是（1）的简称。（2016年上半年）

**解析**：这是缩略语记忆题。VLIW = Very Long Instruction Word。  
**答案方向**：超长指令字。

### 题 102

**原题**：移位指令中的（4）指令的操作结果相当于对操作数进行乘 2 操作。（2016年上半年）

**解析**：逻辑左移一位，相当于数值乘以 2（不考虑溢出）。  
**答案方向**：左移指令。

### 题 103

**原题**：在程序运行过程中，CPU 依据（1）来区分在内存中以二进制编码形式存放的指令和数据。（2016年下半年）

**解析**：同一块存储器里指令和数据形式上都只是二进制，CPU 要靠指令周期不同阶段以及当前控制逻辑来区分。  
**答案方向**：依据**指令周期的不同阶段**。

### 题 104

**原题**：要判字长为 16 位的整数 a 的低四位是否全为 0，则（2）。（2017年上半年）

**解析**：低四位判断一般用按位与掩码 000FH，看结果是否为 0。  
**答案方向**：a AND 000FH = 0。

### 题 105

**原题**：某计算机系统的 CPU 主频为 2.8GHz。某应用程序包括 3 类指令，各类指令 CPI 及比例已知。执行该应用程序时的平均 CPI 和 MIPS 分别为（4）、（5）。（2020年下半年）

**解析**：  

1. 平均 $\text{CPI} = \sum(\text{各类 CPI} \times \text{指令比例})$  
2. $\text{MIPS} = \dfrac{\text{主频}}{\text{平均 CPI} \times 10^6}$  
   若主频单位是 GHz，要先换成 MHz 再代入。  
   **答案方向**：先算平均 CPI，再求 MIPS。
